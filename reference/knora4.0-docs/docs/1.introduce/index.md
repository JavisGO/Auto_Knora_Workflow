## 相关词说明

**本体（Ontology）**——对现实世界的形式化描述，通过将现实中的物体、关联、变化和影响进行抽象，使其能被表示和计算。在企业业务系统中，本体定义了业务领域的概念、实体、属性和关系，使数据从"字段集合"升级为"业务对象网络"。{{aio.name}} 将企业本体分为语义元素和动态元素两个维度。

**实体（Entity）**——对业务中核心要素的定义，一般对应真实世界的具体物体或者抽象概念，由唯一标识和一组属性来表示。例如"客户"、"合同"、"设备"。

**关系（Relation）**——用于描述实体和实体之间的联系，由两个实体和一组属性来表示。例如人和人的亲缘关系，车和人的所属关系。

**事件（Event）**——对业务中产生的一条具体事实的定义，会和一个实体或多个实体相关，可能拥有时间或空间的属性。例如"合同签署记录"、"设备报警记录"。

**行动（Action）**——由语义元素变化来触发，主要是实体的属性、实体间的关系发生了变化、发生了新的事件，从而需要执行某些业务逻辑。Action 是平台将图谱数据操作标准化为可调用接口的核心机制。

**逻辑（Logic / Workflow）**——行动 Action 中的具体执行逻辑，由一组输入和一段处理过程组成。通过将 Agent、Workflow、Tools Set 打通，实现复杂且灵活的执行链路的编排，形成可配置、可复用、可演进的执行单元。

**Knora Claw**——平台内置的 Agentic 自主推理引擎，支持将用户提出的复杂目标自动拆解为可执行的子任务序列，并基于任务特征动态规划执行顺序与所需资源，是平台 Multi-Agent 协同能力的核心载体。

**RAG（Retrieval-Augmented Generation）**——检索增强生成。通过在大模型推理前先从知识库中检索相关内容作为上下文，有效降低模型"幻觉"风险，提升回答的准确性与可追溯性。

**MCP（Model Context Protocol）**——模型上下文协议。平台支持将已构建的智能体以 MCP 协议标准对外发布，使其可被其他支持 MCP 协议的工具或平台调用。

**经纶**——平台内置的结构化数据治理流程引擎，覆盖数据接入、提取、清洗、标准化、标识与融合等关键环节，支持批式与流式双模式处理。

**AIGO**—— AI Generate Ontology，即 AI 生成本体，通过人工智能技术自动识别业务数据中的实体、属性与关系，快速生成知识图谱结构草稿，大幅降低本体建模的门槛与工作量。
