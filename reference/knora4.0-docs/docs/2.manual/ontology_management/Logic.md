# 流程活动（Logic / Workflow）

流程活动（Logic）是 {{aio.name}} 平台的智能体构建核心模块。用户通过低代码、可视化的流程编排引擎，将大语言模型、知识库、工具能力与业务逻辑统一纳入工作流（Workflow）体系，快速定义稳定、可控的业务智能体。

在左侧导航中，流程活动模块入口即 **流程活动（Logic）**，点击后进入 Workflow 列表页。

## Workflow 管理

### 1 创建 Workflow

平台支持以下三种方式创建 Workflow：

**方式一：手动创建**

点击页面右上角的**新建**按钮，在弹窗中填写以下信息：

![手动创建 Workflow](picture/workflow_create.png){ width="100%", loading=lazy }
/// caption
图3-1 手动创建 Workflow
///

- **名称**：Workflow 的中文名称（必填）。
- **图标**：选择一个图标用于列表展示（选填）。
- **描述**：说明该 Workflow 的业务用途（选填）。
- **BOT 应用形态**：选择**聊天式**（多轮对话形式）或**任务式**（一次性输入输出形式）。

填写完成后点击**确认**，系统将创建一个空白 Workflow 并自动跳转至编排画布。

**方式二：基于模板创建**

点击**模板**标签，从平台内置的常见模板中选择，如"基于知识库的对话"等。用户仅需进行少量配置即可完成 Workflow 创建，适合快速构建标准场景的智能体。

**方式三：导入创建**

点击**导入**按钮，上传已导出的 Workflow 配置文件（.zip 格式），系统将解析其中的 .json 配置，自动还原 Workflow 的基本信息、模型参数与编排结构。

!!! note "注意"
    导入的配置文件需来自同版本或兼容版本的 Knora 平台，跨版本导入可能存在节点不兼容的情况。

### 2 Workflow 列表

Workflow 列表以**卡片形式**展示，每张卡片显示 Workflow 的名称、图标、描述、应用形态及创建时间。

![Workflow 列表页](picture/workflow_list.png){ width="100%", loading=lazy }
/// caption
图3-2 Workflow 列表页
///

列表顶部提供两个视图切换：

- **我创建的**：仅显示当前用户创建的 Workflow。
- **分享给我的**：显示其他用户授权分享给当前用户的 Workflow。

**检索**：在搜索框中输入关键字，支持按 Workflow 名称进行实时过滤。

**标签筛选**：点击标签筛选下拉框，选择一个或多个标签，对 Workflow 进行分类过滤，支持多选。

### 3 Workflow 管理操作

在 Workflow 卡片上，点击右上角的**更多操作**按钮（···），可执行以下操作：

| 操作 | 说明 |
|------|------|
| **置顶 / 取消置顶** | 将重要 Workflow 固定展示在列表顶部，再次点击可取消置顶 |
| **编辑** | 修改 Workflow 的图标、名称、BOT 应用类型、可见范围及描述 |
| **复制** | 创建当前 Workflow 的副本，新副本保留原有编排结构，可在副本基础上继续修改，避免重复搭建 |
| **导出配置文件** | 将 Workflow 导出为 .zip 压缩包，解压后的 .json 文件涵盖 Workflow 的基本描述、模型参数、编排配置等完整信息，可导入其他 Knora-AIGO 平台实例 |
| **打标签** | 为 Workflow 添加一个或多个分类标签；标签在组织内共享，所有用户可使用已有标签或创建新标签 |
| **删除** | 删除当前 Workflow，删除后不可恢复，请谨慎操作 |

### 4 Workflow 授权

Workflow 的可见范围与操作权限通过授权机制控制。在 Workflow 详情页或编辑弹窗中，可进行以下配置：

- **可见范围**：选择**本人可见**、**指定用户可见**或**全员可见**。
- **操作权限**：对已授权的用户，可进一步分配**使用权限**（仅可运行）或**管理权限**（可编辑、复制、删除）。
- **密钥授权**：如需通过 API 调用该 Workflow，需将对应密钥授权给此 Workflow（详见第七章 密钥管理）。

## Workflow 流程编排

进入 Workflow 的编排画布后，左侧为节点选择面板，中央为流程画布，右侧为节点配置弹窗（点击节点后展开）。

### 1 画布操作

**整体操作**

画布底部提供全局操作按钮：

- **缩放**：对画布进行放大或缩小，支持"适应画布"将所有节点按比例填满屏幕。
- **自动布局**：将画布中的节点按 DAG 方向自动整理排列。
- **锁定**：锁定画布节点布局，锁定后拖动操作将整体平移画布，避免误操作移动节点。
- **快捷键说明**：点击可查看当前支持的快捷操作说明。

支持的快捷键如下：

| 快捷键 | 功能 |
|------|------|
| `Ctrl + 左键拖拽` | 多选节点 |
| `Shift + 左键拖拽` | 框选区域内的节点 |
| `Delete` / `Backspace` | 删除选中节点 |
| `Ctrl + S` | 保存当前画布 |
| `Esc` | 关闭节点配置弹窗 |

画布支持**随时保存**，保存后在画布顶部显示最新更新时间。

**节点操作**

在画布中，点击任意节点右上角的操作按钮，可执行以下操作：

- **单独运行**（仅适用于大语言模型、知识库检索、工具调用、代码执行节点）：在画布右侧弹出测试窗口，手动输入该节点的输入参数，按运行日志方式展示执行结果，便于效果测试与问题排查。
- **复制节点**：在画布上复制当前节点，生成一个配置相同的新节点，避免重复配置。
- **删除节点**：删除选中节点。删除后需重新连接上下游节点以保证流程连通。
- **帮助**：跳转至该节点类型的操作说明文档。
- **编辑标题**：修改节点在画布上显示的名称，使其更具业务含义，便于在节点较多时快速理解流程结构。

**添加与连接节点**

从左侧节点选择面板将节点拖拽至画布，鼠标悬停节点边缘出现连接点后，拖拽至目标节点即可建立连接，形成 DAG 流程。支持一个节点连接多个上游或下游节点。

### 2 流程节点详解

#### 输入节点

输入节点用于定义 Workflow 的入参，是流程的起始节点，每个 Workflow 有且仅有一个输入节点。

在输入节点配置弹窗中，点击**添加参数**，为每个参数填写：

- **参数名称**：参数的中文显示名称（必填）。
- **参数类型**：支持以下类型：

| 类型   | 说明                                     |
| ---- | -------------------------------------- |
| 文本   | 字符串输入，最常用的参数类型                         |
| 数值   | 数字输入                                   |
| 数组   | 多值输入，用于批量处理场景                          |
| 对象   | 结构化 JSON 对象输入                          |
| 文件   | 支持上传 docx、doc、txt、pdf、xls、xlsx、csv 等格式 |
| 图片   | 支持上传 jpg、jpeg、png 等格式                  |
| 下拉框  | 从预定义选项中选择，需配置选项列表                      |
| 实体引用 | 从已建立的本体中引用所需实体                         |

- **是否必填**：标记该参数在运行时是否必须提供。

对于**聊天式**应用，系统会对参数类型和数量进行自动校验和限制，确保与对话交互形式兼容。

#### 大语言模型（LLM）节点

大语言模型节点用于在流程中调用大模型能力，完成自然语言处理、内容生成、意图识别、推理思考等任务。

从节点面板拖拽**大语言模型（LLM）**节点至画布，连接上下游节点后，点击节点打开配置弹窗：

![LLM 节点配置](picture/llm_node.png){ width="100%", loading=lazy }
/// caption
图3-3 LLM 节点配置
///

**模型**：从平台已配置的模型列表中选择一个用于执行本节点任务的大模型。支持 VL 视觉模型（可解析图片类输入）和推理模型（可生成思维链输出），两类模型在选择列表中有特殊标识。

展开**高级配置**可调整以下参数：

- **温度**：取值 0–1，默认 0.8。值越高，生成结果多样性越强；值越低，输出越稳定。
- **最大推理字符数**：单次推理结果的字符数上限，默认 1024。
- **最大推理时间**：推理允许的最大等待时间（秒），取值 0–120，默认 30。
- **运行错误重试次数**：运行出错后的自动重试次数，默认 1 次。
- **数据错误重试次数**：数据返回异常后的自动重试次数，默认 1 次。

**输入**：从上游节点的输出变量中选择当前节点所需的变量作为模型输入，支持多选。所选变量可在提示词中通过 `@变量名` 方式引用。

!!! note "注意"
    若输入变量中包含图片类参数，只有在**模型**中选择 VL 视觉模型，才能对图片内容进行解析。

**循环参数**：当输入参数中包含数组类型变量时，可指定其中一个数组变量为循环参数，节点将依次对数组中的每个元素调用大模型，最终以数组形式输出所有结果。

**提示词**：在富文本编辑框中撰写提示词，通过 `@` 键引用已选输入变量。支持分别输入**系统提示词**和**用户提示词**，同时支持以问答对形式添加少样本示例。

**输出**：定义节点的输出变量名称与类型，支持添加多个输出。当配置了推理模型时，节点默认附带**思维链**输出，可被下游节点引用。

#### 知识库检索节点

知识库检索节点用于在流程中对知识库内容进行检索匹配，是构建 RAG 智能问答系统的核心节点。

从节点面板拖拽**知识库检索**节点至画布，点击节点打开配置弹窗：

![知识库检索节点配置](picture/kb_rag_node.png){ width="100%", loading=lazy }
/// caption
图3-4 知识库检索节点配置
///

**查询条件**：从上游节点输出变量中选择用作检索输入的变量，支持多选。

**知识库配置**：从平台已构建的知识库中选择一个或多个作为检索范围。支持两类知识库：

- **文档知识库**（基础文档 / 专业文档）：检索时可配置检索模式（语义检索/全文检索/混合检索/交集检索）、引用上限、文档引用上限。
- **数据知识库**：对结构化数据表中指定的向量化字段进行相似度匹配。

#### 本体数据推理节点

本体数据推理节点用于在流程中直接查询平台本体图谱数据，实现基于业务语义的实体检索与关系探查。

![本体数据推理节点配置](picture/ontology_node.png){ width="100%", loading=lazy }
/// caption
图3-5 本体数据推理节点配置
///

**查询方式**支持两种模式：

- **自然语言查询**：以自然语言输入查询意图，系统通过 Text2Query 能力自动转换为图谱查询语句。
- **精确查询**：直接配置查询条件，支持 Text2SQL、条件查询、检索查询、实体查关系四种方式。

#### Action 执行节点

Action 执行节点用于在流程中触发已在本体管理中定义的 Action，对图谱数据执行写操作。

从节点面板拖拽 **Action 执行**节点至画布，点击节点打开配置弹窗：

- **选择 Action**：从已发布的 Action 列表中选择需要执行的 Action。
- **参数映射**：根据所选 Action 的参数定义，配置每个参数的取值来源。

#### 工具调用节点

工具调用节点用于在流程中调用平台已集成的外部工具，扩展智能体的执行能力。

从节点面板拖拽**工具调用**节点至画布，点击节点打开配置弹窗：

- **选择工具**：从平台的自定义工具、内置工具、智能体工具及 MCP 工具列表中选择。
- **参数配置**：根据工具的输入参数定义进行映射。
- **输出**：定义工具调用结果的输出变量。

#### 寄存器节点

寄存器节点用于在流程执行过程中存储和管理上下文变量，实现跨节点的信息传递。

**存储模式**支持三种：

- **一次性**：存储后只能被读取一次，读取后自动清除。
- **追加**：每次写入时将新内容追加至已有内容之后。
- **替换**：每次写入时用新内容完全覆盖原有内容。

#### 条件选择节点

条件选择节点（If-Else）用于在流程中根据判断条件将执行路径分支。

- 可添加多个 **If** 分支，每个分支配置一组判断条件。
- 条件不满足时，流程走 **Else** 分支。

#### 代码执行节点

代码执行节点允许用户在流程中编写和执行自定义脚本。

- **编程语言**：选择 **Python** 或 **JavaScript**。
- **输入**：定义脚本所需的输入变量。
- **输出**：定义输出变量名称与类型。

#### 变量聚合节点

变量聚合节点用于将来自不同条件分支的输出变量合并为统一变量。

#### 循环节点

循环节点用于对数组类数据进行批量处理。

**循环模式**支持三种：

- **数组循环**：指定一个数组类变量，对数组中每个元素依次执行循环体内的子流程。
- **指定次数**：设定循环执行的固定次数。
- **无限循环**：持续执行，直到在循环体内通过条件判断触发跳出。

#### 知识库写入节点

知识库写入节点用于在流程执行过程中，将生成或处理后的内容动态写入平台知识库。

- **目标知识库**：选择需要写入的文档知识库或数据知识库。
- **写入内容**：从上游节点的输出变量中选择需要写入的内容字段。

#### 数据集节点

数据集节点用于在流程中对平台已配置的数据集进行自然语言查询。

- **数据集**：选择需要查询的数据集。
- **查询输入**：从上游节点输出变量中选择用户的自然语言查询内容。

#### 批处理节点

批处理节点用于对数组类输入数据启动并行子工作流处理，相比循环节点的顺序执行，批处理节点支持多个子流程同时并行运行。

#### 输出节点

输出节点定义 Workflow 的最终返回内容，是流程的终止节点，每个 Workflow 有且仅有一个输出节点。

**支持的输出类型**：

| 输出类型 | 说明 |
|------|------|
| 文本 | 纯文本内容输出，支持 Markdown 渲染 |
| 数值 | 数字类结果输出 |
| 数组 | 多值结果输出 |
| 对象 | 结构化 JSON 数据输出 |
| 文件 | 文件类内容输出，支持用户下载 |
| 关系图 | 图谱关系可视化输出 |
| 地理坐标 | 地图位置数据输出 |
| 二维图表 | ECharts 图表输出 |
| 消息推送 | 触发平台内部通知消息推送 |
| 表格 | 结构化表格数据输出 |
| 引用文档 | 知识库检索引用文档的溯源信息输出 |
| 思维链 | 推理模型的思考过程实时输出 |
| 预定义选项 | 将输出内容定义为可直接点击的快捷选项 |

### 3 全局功能与系统变量

**开场白配置**

在画布左侧的**全局功能**面板中，可配置 Workflow 的开场白：

- **开场白标题**：智能体对话界面顶部显示的名称文案。
- **对话开场白**：用户进入对话界面时，智能体主动展示的引导内容。

**系统变量**

- **当前用户**：自动取当前登录用户的用户名信息。
- **当前时间**：自动取 Workflow 执行时的系统时间。
- **当前域名**：自动取当前平台部署的域名信息。

## Workflow 调试与发布

### 1 调试运行

在编排画布右上角点击**运行**按钮，画布右侧展开**运行态侧边栏**：

在侧边栏中输入测试参数，点击**调试运行**，系统将执行 Workflow 并在画布中实时展示各节点的运行状态与结果。

**版本管理**

- **仅保存**：只保存当前的快照。
- **保存新版本**：新增一个版本。当存在多个版本时，可以选择某一版本进行发布。

### 2 发布 Workflow

完成调试后，点击画布右上角的**发布**按钮，弹出发布方式选择页，支持以下发布形式：

**发布为 MCP 工具**：将 Workflow 以 MCP（Model Context Protocol）协议标准对外发布，生成符合 MCP 协议规范的配置信息与调用接口。

**发布为 API**：将 Workflow 发布为可供第三方系统调用的 API 接口，平台自动生成 API 说明文档。

**网页嵌入**：将 Workflow 的运行态对话界面生成 **iframe 代码**，可嵌入第三方系统页面。
