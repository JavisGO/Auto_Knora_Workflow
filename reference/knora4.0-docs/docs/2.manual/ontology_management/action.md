# 业务操作与行为（Action）管理

业务操作与行为（Action）将业务操作标准化为可调用的执行接口，支持对图谱数据进行增删改操作，或触发业务流程逻辑。Action 是实体看板中"操作按钮"的底层定义，也是流程编排中"Action 执行节点"的调用来源。

在左侧导航中点击**业务操作与行为（Action）**，进入行为管理列表页。

## 1 新建行为

点击页面右上角的**新建行为**按钮，进入行为创建页面，分为两个步骤：

**第一步：基本信息**

![新建行为 - 基本信息](picture/action_create_step1.png){ width="100%", loading=lazy }
/// caption
图2-9 新建行为 - 基本信息
///

填写以下信息：

- **行为名称**：Action 的中文名称，将展示在实体看板的操作按钮上（必填）。
- **行为标识符（API Name）**：Action 的唯一英文标识，用于接口调用（必填）。
- **图标**：为 Action 选择一个图标（选填）。
- **标签**：为 Action 添加分类标签，便于检索（选填）。
- **描述**：说明该 Action 的业务含义与适用场景（选填）。

填写完成后，点击**下一步**。

**第二步：规则配置**

规则配置用于定义 Action 的具体执行逻辑，支持添加多个规则步骤，按顺序依次执行。

![新建行为 - 规则配置](picture/action_create_step2.png){ width="100%", loading=lazy }
/// caption
图2-10 新建行为 - 规则配置
///

点击**添加步骤**，每个步骤需配置以下内容：

**操作类型**，支持以下六种：

| 操作类型        | 说明               |
| ----------- | ---------------- |
| 添加实体        | 向图谱中新增一个实体实例     |
| 编辑实体        | 修改指定实体实例的属性值     |
| 删除实体        | 从图谱中删除指定实体实例     |
| 添加/删除关系     | 在两个实体之间新增或删除关联关系 |
| 添加事件        | 向图谱中新增一条事件记录     |
| 调用逻辑（Logic） | 触发一个已定义的Logic执行  |
| 消息通知        | 向指定用户或用户组发送通知消息  |

**参数映射（Input Mapping）**，为步骤中所需的参数指定取值来源，支持以下方式：

- **用户输入**：Action 执行时由用户在弹窗表单中手动填入。
- **参数引用**：引用当前 Action 前序步骤的输出值或上下文参数。
- **固定值**：配置阶段直接指定的静态值，执行时不可修改。
- **当前用户**：自动取当前执行该 Action 的登录用户信息。

**执行结果需用户确认（高风险操作）**：对于删除实体、删除关系等不可逆操作，可开启二次确认弹窗，要求用户在执行前手动确认，防止误操作。

全部步骤配置完成后，点击**保存**完成 Action 创建。

## 2 编辑行为

在行为列表中，点击目标 Action 名称进入详情页，点击**编辑**按钮，可对 Action 的基本信息与规则配置进行修改。

## 3 标签管理与行为检索

操作方式与实体管理一致，请参考 **实体标签管理** 与 **实体检索**。
