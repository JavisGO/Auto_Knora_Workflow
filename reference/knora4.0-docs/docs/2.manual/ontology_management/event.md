# 事实明细（Event）管理

事实明细（Event）是对业务中产生的具体事实的定义，用于记录业务对象的状态变化与行为轨迹。事件会关联一个或多个实体，可能拥有时间或空间的属性，是图谱中时序与动态分析的基础数据单元。

在左侧导航中点击**事实明细（Event）**，进入事件管理列表页。

## 1 新增事件

点击页面右上角的**新建事件**按钮，进入事件创建向导。步骤与实体创建一致（基本信息 → 数据映射 → 属性定义 → 属性映射），以下重点说明事件特有的配置项。

**第一步：基本信息**

![新增事件 - 基本信息](picture/event_create_step1.png){ width="100%", loading=lazy }

填写以下信息：

- **事件名称**：事件类型的中文名称（必填）。
- **事件标识符（API Name）**：事件在系统内的唯一英文标识（必填）。
- **主体实体类型**：该事件所关联的主体实体类型，即"是谁发生了这个事件"（必填）。
- **事件描述**：对事件的发生背景与触发含义进行说明（选填）。

其余步骤操作方式与实体创建相同，请参考 **新增实体**中的对应步骤说明。
