# 自主推理（Knora Claw）

Knora Claw 是 {{aio.name}} 平台内置的 Agentic 自主推理引擎，支持将用户提出的复杂目标自动拆解为可执行的子任务序列，并基于任务特征动态规划执行顺序与所需资源。平台采用 Multi-Agent 架构，支持多个智能体之间的自主协作与嵌套调用，根据用户意图选择最合适的 Agent 参与执行，解决难以通过单一规则或固定流程表达的复杂业务问题。

在左侧导航中点击 **Knora Claw**，进入 Knora Claw 列表页。

## 1 创建 Knora Claw

点击页面右上角的**新建**按钮，在弹窗中填写以下信息：

![创建 Knora Claw](picture/claw_create.png){ width="100%", loading=lazy }
/// caption
图4-1 创建 Knora Claw
///

- **名称**：Knora Claw 的中文名称（必填）。
- **图标**：选择一个图标用于列表展示（选填）。
- **描述**：说明该 Knora Claw 的业务用途（选填）。

填写完成后点击**确认**，系统创建 Knora Claw 并进入配置页面。

支持以**导入**方式创建：点击导入按钮，上传已导出的 Knora Claw 配置文件，系统自动还原其配置结构。

## 2 Knora Claw 列表与检索

Knora Claw 列表以**卡片形式**展示，每张卡片显示名称、图标、描述及应用形态。

列表支持以下两个视图切换：

- **我创建的**：仅显示当前用户创建的 Knora Claw。
- **分享给我的**：显示其他用户授权分享给当前用户的 Knora Claw。

在搜索框中输入关键字，可通过名称对 Knora Claw 进行实时过滤检索。

## 3 Knora Claw 管理操作

在 Knora Claw 卡片上，点击右上角的**更多操作**按钮（···），可执行以下操作：

| 操作 | 说明 |
|------|------|
| **编辑** | 修改 Knora Claw 的图标、名称及描述信息 |
| **配置** | 进入到 Claw 的配置页面 |
| **置顶 / 取消置顶** | 将 Knora Claw 固定展示在列表顶部 |
| **授权** | 配置该 Knora Claw 的可见范围及操作权限 |
| **复制** | 创建当前 Knora Claw 的副本 |
| **导出** | 将 Knora Claw 配置导出为文件，用于在其他平台实例中导入复用 |
| **删除** | 删除当前 Knora Claw，删除后不可恢复 |

## 4 Knora Claw 运行

点击 Knora Claw 卡片或进入详情页，点击**运行**按钮，进入 BOT 对话界面。

### 对话交互

用户在输入框中输入问题或指令，Knora Claw 将根据其配置的定义与工具链返回答案，并支持基于上下文和执行模式的理解进行连续多轮对话。

输入框支持以下附件类型的上传：

- **文件**：上传文档类文件（docx、pdf、txt、xlsx 等）作为对话上下文。
- **图片**：上传图片文件（jpg、jpeg、png 等）供视觉模型解析。
- **音频**：上传音频文件，由语音识别模块转换为文本后进行处理。

### 会话管理

- 点击**新建会话**，开启一次全新的独立对话，不携带历史上下文。
- 点击左侧**历史记录**，可查看并切换至此前的对话会话。

### 消息操作

对每条 Knora Claw 的回复，支持以下操作：

- **点赞 / 点踩**：对回复质量进行反馈评价。
- **复制**：将回复内容复制至剪贴板。
- **重新提问**：清除当前回复，重新触发对该条输入的推理执行。

### 推理日志

在对话界面中，可展开 Knora Claw 的**推理日志**，查看本次推理的完整执行链路，包括：

- 任务拆解步骤
- 各子任务调用的工具或 Workflow
- 各步骤的输入输出内容

便于理解推理过程与排查异常。

### 输出能力

Knora Claw 支持多种形式的内容输出：

- **流式输出**：回复内容实时逐字生成，提升交互响应体验。
- **推理过程输出**：当配置了推理模型时，可实时展示模型的思维链过程。
- **导出 PDF**：将当前对话内容导出为 PDF 文件。
- **HTML 展示**：将回复内容以 HTML 格式渲染展示。
- **文件下载**：对于 Knora Claw 生成的文件类输出，支持直接下载。

### 推理链路可视化与进度展示

在推理执行过程中，界面顶部展示：

- 当前执行的**计划进度条**：实时查看整体任务的完成进度。
- 消耗的**上下文窗口**使用量：便于掌握会话容量状态。
