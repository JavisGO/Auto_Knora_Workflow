# 业务关联（Relation）管理

业务关联（Relation）用于描述实体和实体之间的联系，明确业务对象之间的关联脉络与语义方向。例如"客户—签署—合同"、"员工—归属—部门"。关系定义是知识图谱中连接业务对象、实现语义推理的重要基础。

在左侧导航中点击**业务关联（Relation）**，进入关系管理列表页。

## 1 新增关系

点击页面右上角的**新建关系**按钮，进入关系创建向导，同样分为四个步骤（基本信息 → 数据映射 → 属性定义 → 属性映射），步骤逻辑与实体创建一致，以下重点说明关系特有的配置项。

**第一步：基本信息**

![新增关系 - 基本信息](picture/relation_create_step1.png){ width="100%", loading=lazy }

填写以下信息：

- **关系名称**：关系类型的中文名称（必填）。
- **关系标识符（API Name）**：关系在系统内的唯一英文标识（必填）。
- **来源实体类型**：该关系的起始端实体类型（必填）。
- **目标实体类型**：该关系的终止端实体类型（必填）。
- **关系方向**：选择有向或无向关系。
- **关系描述**：对该关系的业务含义进行说明（选填）。

其余步骤（数据映射、属性定义、属性映射）操作方式与实体创建相同，请参考 **2.2.1 新增实体**中的对应步骤说明。
